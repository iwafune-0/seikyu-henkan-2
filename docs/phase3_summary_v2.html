📦 実施内容

ステップ1: プロジェクト初期化

npm create vite@latest frontend -- --template react-ts cd frontend npm install

技術スタック （使用する技術の組み合わせ）:

  • Vite 7.1.9 - 開発を高速にするツール
  • React 19.2.0 - 画面を作る人気の技術
  • TypeScript 5.9.3 - 型付きのプログラミング言語、バグを減らせる

→ Vite （ヴィート） というツールで、React + TypeScriptのプロジェクトを作成しました

ステップ2: Tailwind CSS セットアップ

npm install -D tailwindcss@^3.4.18 postcss autoprefixer npx tailwindcss init -p

💡 重要な決定: Tailwind CSS v4からv3.4.18にダウングレード

理由: Tailwind CSS v4 （最新版） でPostCSS plugin （CSSを処理するツールの拡張機能） の破壊的変更 （互換性を壊す大きな変更） があり、border-border 等のユーティリティクラス （便利な部品） がエラーになったため。

v3.4.18を使用することで、shadcn/ui （部品集） との互換性と安定性を確保しました。

設定内容:

  • CSS変数ベースのテーマ設定 （色やサイズを一括管理）
  • プライマリカラー （メインの色）: エメラルドグリーン（#10b981）
  • shadcn/ui互換の色設定

→ Tailwind CSSというデザインツールをインストールして、綺麗な見た目にしました

ステップ3: 依存パッケージインストール

npm install @supabase/supabase-js zustand react-router-dom lucide-react npm install -D @types/node

主要パッケージ （必要なライブラリ）:

パッケージ バージョン 用途
@supabase/supabase-js 2.74.0 Supabase認証・データベース （ログイン機能とデータ保管）
zustand 5.0.8 状態管理 （ログイン状態などを管理）
react-router-dom 7.9.3 ルーティング （ページ間の移動）
lucide-react latest アイコン （ボタンや見出しの絵文字）
clsx latest クラス名結合 （CSSクラスを便利に組み合わせる）
tailwind-merge latest Tailwindクラス結合 （重複を自動解決）

→ 必要なライブラリ（便利な道具）を全てインストールしました

ステップ4: パス alias 設定

vite.config.ts と tsconfig.app.json で設定：

alias: { '@': path.resolve(__dirname, './src'), }

設定内容:

  • @/ で src/ を参照可能に
  • 例: import { Button } from '@/components/Button'
  • 開発サーバーのポート （扉番号） を 5174 に設定

→ ファイルを読み込むときに、@ という短縮記号を使えるようにしました

ステップ5: shadcn/ui パターン実装

lib/utils.ts に便利な関数を作成：

import { type ClassValue, clsx } from "clsx" import { twMerge } from "tailwind-merge" export function cn(...inputs: ClassValue[]) { return twMerge(clsx(inputs)) }

→ cn() という関数で、CSSクラスを便利に組み合わせられるようにしました

ステップ6: Supabase クライアント初期化

lib/supabase.ts でSupabaseと接続：

import { createClient } from '@supabase/supabase-js' const supabaseUrl = import.meta.env.VITE_SUPABASE_URL const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY export const supabase = createClient(supabaseUrl, supabaseAnonKey)

環境変数 （秘密情報を安全に管理） (.env):

VITE_SUPABASE_URL=https://mock-project.supabase.co VITE_SUPABASE_ANON_KEY=mock-anon-key

💡 Phase 3では「モック」を使用

Phase 3の時点ではまだSupabaseバックエンド （裏側の処理） を構築していないため、仮のURL（mock-project.supabase.co）を設定して、フロントエンド単体で動作確認できるようにしました。

Phase 5（環境構築）で本物のSupabaseに切り替えます。

ステップ7: 型定義（types/index.ts）

データの形を定義する「型」を作成：

export interface User { id: string email: string role: 'admin' | 'user' created_at: string } export interface LoginRequest { email: string password: string } export interface LoginResponse { user: User token: string }

→ ユーザー情報やログイン時のデータの形を定義して、型チェック （データの形が正しいか自動確認） できるようにしました

💬 ユーザーフィードバックと改善

改善1: バリデーション改善

ユーザーからの要望:

「このフィールドを入力してください」というポップアップ （ブラウザの警告） が何か所か出ているのですが、「パスワードは8文字以上で入力してください」のような形式でテキストで表示していただけると見やすい

対応:

  • 全ての required、minLength、type="email" 属性 （HTMLの自動チェック） を削除
  • カスタムJavaScript （独自の） バリデーション （入力チェック） に置き換え
  • エラーメッセージを setError() でテキスト表示

✅ ブラウザのポップアップではなく、画面内に親切なエラーメッセージを表示するようにしました

改善2: パスワード要件の強化

変更履歴:

  1. 初期要件: 8文字以上
  2. 強化案1: 英字・数字・記号を含む8文字以上
  3. 最終決定: 英字・数字を含む8文字以上（記号は不要）

バリデーションロジック （入力チェックの処理）:

// 8文字以上 if (password.length < 8) { setError('パスワードは8文字以上で入力してください。') return } // 英字を含む if (!/[a-zA-Z]/.test(password)) { setError('パスワードには英字を含めてください。') return } // 数字を含む if (!/[0-9]/.test(password)) { setError('パスワードには数字を含めてください。') return } // 半角英数字のみ if (!/^[a-zA-Z0-9]+$/.test(password)) { setError('パスワードは半角の英字・数字のみで入力してください。') return }

✅ セキュリティとユーザビリティのバランスを考えた、適切なパスワードルールにしました
